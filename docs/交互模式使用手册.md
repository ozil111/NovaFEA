# hyperFEM 交互命令行使用手册

基于 `CommandProcessor.cpp` 实现。

## 交互模式入口

- **启动方式** 不带 `--input-file` 参数启动 `hyperFEM_app` 时，会进入交互模式。提示符：`hyperFEM>`，输入一行命令回车执行。
- **基础约定** 多数命令都要求已 **加载 mesh**（`session.mesh_loaded == true`），否则会报错。拓扑相关命令要求已先执行 `build_topology`。`help` 可随时查看支持的命令列表。

## 基础命令

### 退出与帮助

- **退出程序** `quit` / `exit`
- **查看命令列表** `help`

## 网格导入 / 导出

### 通用 XFEM / JSON 导入

- **`import`** `import <path_to_file>`支持扩展名：`.xfem`、`.json`、`.jsonc`。 会先清空当前数据，再调用相应解析器。 导入成功后打印节点/单元数量。

### Simdroid 项目导入

- **`import_simdroid`** `import_simdroid <path_to_control.json>`自动假定 `mesh.dat`与 `control.json`在同一目录。 `mesh.dat`路径：`<control_dir>/mesh.dat` 成功后： `session.mesh_loaded = true` 立即构建 `SimdroidInspector`索引（支持后续 `node`/ `elem`/ `list_parts`等命令）。

### Simdroid 项目导出（Blueprint Strategy）

- **`export_simdroid`** `export_simdroid <output_dir | mesh.dat | control.json> [control.json]`**形式 1**：显式指定两个文件： `export_simdroid <mesh.dat> <control.json>`**形式 2**：只给一个参数，自动推导： 给 `.json`/ `.jsonc`：该文件视为 `control.json`，`mesh.dat`写到同级目录。 给 `.dat`：该文件视为 `mesh.dat`，`control.json`写到同级目录。 给目录：在目录下写 `mesh.dat`和 `control.json`。 导出依据当前 ECS（包括你通过交互命令改动后的内容）。

### 传统 XFEM 导出

- **`save`** `save <path_to_output_file.xfem>`将当前 mesh 数据导出为 XFEM 文本格式。

## Mesh 信息与拓扑分析

### 概览信息

- **`info`** `info`若无 mesh：打印 “No mesh loaded.” 若有 mesh： 节点数：`Component::Position` 单元数：`Component::Connectivity` set 数：`Component::SetName` 若拓扑已构建：打印面数、实体 body 数量。

### 拓扑构建与连通体

- **`build_topology`** `build_topology`使用 `TopologySystems::extract_topology`从单元连接性生成拓扑。 存入 `TopologyData`（放在 `registry.ctx()`中）。 成功后 `session.topology_built = true`。
- **`list_bodies`** `list_bodies`依赖已构建的拓扑。 调用 `TopologySystems::find_continuous_bodies`，打印每个连续体的 ID 与包含的单元数。
- **`show_body`** `show_body <body_id>`列出指定 body 中所有单元的原始 ID（`Component::OriginalID`），逗号分隔。

## Simdroid 部件与图分析

### 部件（Part）相关

- **`list_parts`** `list_parts`依赖 `SimdroidInspector`已构建（Simdroid 导入后自动构建）。 打印各 Part： Part 名称 材料名（若有） 单元数量
- **`delete_part`** `delete_part <part_name1> [part_name2 ...]`每次删除前会重建 inspector 索引，保证查找表有效。 对每个 Part： 删除该 Part 下的单元、孤立节点、关联 surface、失效的接触/约束等（细节在 `SimdroidInspector::delete_part`中）。 删除后再次重建 inspector，清空拓扑缓存、重置 `session.topology_built = false`。

### 部件连通性图（Graph）

- **`graph`** `graph [output_filename]`默认输出 `connectivity.html`。 步骤： 使用 `GraphBuilder::build`构建 Part 图。 统计孤立部件数量。 调用 `MermaidReporter::generate_interactive_html`生成交互式 HTML 报告。 在 Windows 上尝试自动用浏览器打开。

### 约束 / 接触一致性检查

- **`validate_constraints`** `validate_constraints`要求已通过 `import_simdroid` 成功加载 Simdroid 模型。 重新调用 `SimdroidParser::validate_contacts`、`validate_rigid_bodies`、`validate_cross_constraints` 对当前 ECS 中的接触与刚体/约束进行一致性检查： 检测接触 master/slave 集合是否引用有效的 set，是否存在节点同时出现在同一接触的 master/slave； 检测刚体 master/slave 节点重叠； 检测同一节点在不同定义中同时作为 master 与 slave（如 Contact 与 RigidBody/MPC 交叉）。 所有问题以 `spdlog::warn` 方式打印，可在多次编辑后随时手动复查。
- **`list_constraint_warnings`** `list_constraint_warnings`打印“跨约束 master/slave 冲突”的**详细节点列表**（导入或最近一次 `validate_constraints` 运行时缓存的结果）。 对每个冲突节点输出：`NodeID` 以及它分别在哪些定义中作为 master / slave（例如 `Contact:*`、`RigidBody:*`、`MPC:*`）。 若无缓存则提示先执行 `import_simdroid` 或 `validate_constraints`。

## 节点 / 单元基础操作（增删改）

> **注意**：这些命令都是对 ECS 里 NodeID / ElementID / Connectivity 的直接操作，会影响后续导出与求解。其中拓扑结构变化时会自动清空拓扑缓存并重建 inspector。

### 节点

- **查看节点** `node <nid>`通过 `SimdroidInspector`： 打印坐标 被哪些单元使用 所属 Part 列表
- **新增节点** `node_add <nid> <x> <y> <z>`**要求**： 已有 mesh。 `nid`不得重复（通过 `Component::NodeID`检查）。 **行为**： 创建带 `Position`/ `NodeID`/ `OriginalID`的节点实体。 使拓扑失效，重建 inspector 索引。
- **移动节点（只改坐标）** `node_move <nid> <x> <y> <z>`若找不到节点则报错。 仅修改 `Position`，不更改连通信息： 不触发拓扑重建。 不重建 inspector（映射还是同一个节点实体）。
- **删除节点** `node_delete <nid>`**安全策略**： 扫描所有单元的 `Connectivity.nodes`，若节点仍被任一单元使用，则拒绝删除并提示对应 `eid`。 只有完全未被使用的节点才允许删除。 **删除成功后**： 清空拓扑缓存。 重建 inspector。

### 单元

- **查看单元** `elem <eid> element <eid>`通过 `SimdroidInspector`： 打印单元类型 ID（`ElementType.type_id`） 所属 Part 所含节点的 NodeID 列表。
- **新增单元** `elem_add <eid> <nid1> <nid2> ...`**要求**： 已有 mesh。 至少 2 个节点 ID。 `eid`不得重复。 所有 `nid`必须存在。 **行为**： 根据节点数自动推断 `ElementType.type_id`： 2 → 102 (Line2) 3 → 203 (Tri3) 4 → 304 (Tet4 / Quad4，按体单元处理) 8 → 308 (Hex8) 10 → 310 (Tet10) 20 → 320 (Hex20) 创建带 `ElementID`/ `OriginalID`/ `ElementType`/ `Connectivity`的单元实体。 清空拓扑缓存，重建 inspector。
- **删除单元** `elem_delete <eid>`**删除前的清理**： 找到所有 `SurfaceParentElement.element`== 该单元 的 `surface`： 从所有 `SurfaceSetMembers`中移除这些 `surface`。 销毁这些 `surface`实体。 从所有 `ElementSetMembers`中移除该单元实体。 然后销毁单元实体本身。 之后会清空拓扑缓存并重建 inspector。

## 通用检查面板（Panel）

- **`panel`** `panel <type> <id_or_name>`在终端中打开一个 **FTXUI 组件化面板**，以视窗形式查看单个实体的各组件数据。 显示完毕后会提示 “Press Enter to return...”，按 **Enter** 返回命令提示符。

**类型与参数**：

| `type`       | 含义     | `id_or_name` 说明 |
|-------------|----------|-------------------|
| `node`      | 节点     | 节点 ID（整数）   |
| `elem` / `element` | 单元 | 单元 ID（整数）   |
| `part`      | 部件     | 部件名称（字符串）|
| `set`       | 集合     | 集合名称（字符串）|

**前置条件**： 已通过 `import_simdroid` 加载 mesh； 查看 **node** 或 **elem** 时，会先自动构建/使用 `SimdroidInspector` 索引（若尚未构建会先 “Building inspection index...”）。 若 `type` 或 `id_or_name` 找不到对应实体，会报错并提示 “not found”。

**面板内容**： 顶部为 “hyperFEM Universal Inspector” 及实体 ID； 中间为多个 **组件视窗**（如 Position、NodeID、Connectivity、SimdroidPart、Force path 等），仅显示该实体实际拥有的组件； 对 **Node** 会额外显示力路径信息（是否 load point / constraint point、所属 Part、Part 间连接）； 对 **Element** 会显示 “Contains Nodes” 及可用的 node 列表； 底部为操作提示 “ [P] Panel  [G] Graph  [Q] Quit ”（当前版本仅支持按 Enter 退出）。

**示例**： `panel node 12`、`panel element 1`、`panel part Component_2_HMprop_Part-2-1_Set-1`、`panel set FixNodes`。

## Set（集合）操作

> **注意**：Set 本质上是一个实体，带 `SetName`，并可选挂 `NodeSetMembers`/ `ElementSetMembers`/ `SurfaceSetMembers`。

### 列出所有 set

- **`list_sets`** `list_sets`遍历所有 `SetName`： 检查是否有 `NodeSetMembers`/ `ElementSetMembers`/ `SurfaceSetMembers`： 只挂 Node：`type=node` 只挂 Element：`type=element` 只挂 Surface：`type=surface` 任意组合：`type=mixed` 只挂 `SetName`：`type=generic` 打印名称、推断类型和成员数量（若是纯 node/element/surface 集）。

### 查看某个 set 详细成员

- **`set_info`** `set_info <set_name>`若 set 不存在：报错。 若有 `NodeSetMembers`： 打印所有成员节点的 NodeID（分多行，每行最多 32 个）。 若有 `ElementSetMembers`： 打印所有成员单元的 ElementID。

### 向 set 添加节点 / 单元

- **`set_addnode`** `set_addnode <set_name> <nid1> [nid2 ...]`如 set 不存在，会新建一个仅带 `SetName`的实体。 挂上或复用 `NodeSetMembers`。 将指定 NodeID 对应的节点实体加入 members（不重复添加）。
- **`set_addelem`** `set_addelem <set_name> <eid1> [eid2 ...]`同上，但针对 `ElementSetMembers`+ `ElementID`。

### 从 set 中移除节点 / 单元

- **`set_removenode`** `set_removenode <set_name> <nid1> [nid2 ...]`要求 set 存在并带 `NodeSetMembers`。 对每个给定 `nid`，按实体删除所有匹配项。
- **`set_removeelem`** `set_removeelem <set_name> <eid1> [eid2 ...]`要求 set 存在并带 `ElementSetMembers`。 同理按实体删除。

## 推荐使用顺序示例

1. **加载并检查模型** `import_simdroid case/hex8r_mat1/control.json info list_parts list_sets`
2. **局部修改 mesh** `node_add 999 0.0 0.0 0.0 elem_add 888 1 2 3 4 5 6 7 8 set_addelem Part_A 888 set_addnode FixNodes 999`
3. **拓扑分析与导出** `build_topology list_bodies graph export_simdroid out_project save out_mesh.xfem`
