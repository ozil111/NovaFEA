# 实验性开发流程：从 JAX 数学原型到高性能 C++ 求解器集成

## 概述

在开发高性能数值求解器（如有限元方法 FEM 的本构模型、非线性方程求根算法）时，算法工程师通常面临两难境地：

1. **Python/JAX** 表达数学公式极其优雅，自动微分（AutoDiff）让验证变得简单，但难以无缝嵌入到对性能和内存要求极高的原生 C++ 求解器中。
2. **纯 C++** 运行极快，但手动推导复杂能量泛函的一阶导数（应力）和二阶导数（切线刚度矩阵/Hessian），不仅耗时、极易出错，且难以进行深度优化。

本工作流提出了一种**“Python 原型验证 -> 符号图优化与代码生成 -> C++ 零依赖集成”**的降维开发模式。该流程将数学推导与底层工程解耦，最大化开发效率与执行性能。

------

## 阶段一：JAX 原型验证 (Ground Truth 确立)

**目标：** 快速验证物理和数学公式的正确性，利用 JAX 的自动微分机制，生成可信的数值对照基准（Ground Truth）。

**操作步骤：**

1. 使用 `jax.numpy` 编写能量密度函数 $\Psi(F)$。
2. 使用 `jax.grad` 自动生成一阶偏导（例如第一类 Piola-Kirchhoff 应力 $P$）。
3. 使用 `jax.hessian` 自动生成二阶偏导（例如材料的切线刚度矩阵 $\frac{\partial^2 \Psi}{\partial F \partial F}$）。
4. 构建单元测试，输入随机的形变梯度测试用例，记录输出结果。

**示例：**

```Python
import jax
import jax.numpy as jnp

def neo_hookean_energy(F, mu, bulk):
    J = jnp.linalg.det(F)
    C = F.T @ F
    I1 = jnp.trace(C)
    return 0.5 * mu * (I1 - 3.0) - mu * jnp.log(J) + 0.5 * bulk * (J - 1.0) ** 2

# JAX 自动微分作为验证基准
get_pk1_stress = jax.grad(neo_hookean_energy, argnums=0)
```

*注：此阶段的代码不考虑运行效率，仅用于确保数学逻辑 100% 正确。*

## 阶段二：SymPy 符号推导与代码生成 (Core Magic)

**目标：** 将已验证的数学逻辑转化为脱离 Python 运行时的底层代码，并利用编译器级别的数学优化榨干性能。

**操作步骤：**

1. **符号化映射：** 使用 `sympy` 重新构建能量函数，将输入变量（如 $3 \times 3$ 矩阵 $F$）展开为 1D 数组形式，以适配 C++ 的连续内存指针访问。
2. **解析求导：** 调用 `sympy.diff` 获取精确的解析偏导表达式。相比于 JAX 的数值计算图，这里得到的是纯代数公式。
3. **公共子表达式消除 (CSE)：** 【关键步骤】调用 `sympy.cse` 对庞大的偏导数矩阵进行全局扫描。CSE 算法会提取出所有重复计算的中间项（如行列式 $J$、伴随矩阵元素），将其定义为局部变量。
4. **C++ 源码生成：** 使用 `sympy.ccode` 将优化后的抽象语法树（AST）直接打印为合法的 C/C++ 源码。

**优势：**

- 彻底消除了复杂求导过程中的人为推导错误。
- CSE 优化后的代码，其执行效率通常**优于**手写 C++ 代码，因为人类难以在包含数十个项的公式中找出所有最佳的公共乘数。

```python
import sympy as sp

def generate_pk1_pk2_cpp():
    # 1. 定义标量参数
    mu, bulk = sp.symbols('mu bulk', real=True)
    
    # 2. 定义形变梯度矩阵 F (3x3 展开为 1D)
    F_syms = [sp.Symbol(f'F[{i}]', real=True) for i in range(9)]
    F = sp.Matrix(3, 3, F_syms)
    
    # 3. 运动学量计算
    J = F.det()
    C = F.T * F
    I1 = sp.trace(C)
    
    # 4. Neo-Hookean 能量密度
    energy = 0.5 * mu * (I1 - 3.0) - mu * sp.log(J) + 0.5 * bulk * (J - 1.0)**2
    
    # 5. 求 PK1 应力 (P)
    P = sp.Matrix(3, 3, lambda i, j: sp.diff(energy, F[i, j]))
    
    # 6. 求 PK2 应力 (S): S = F^{-1} * P
    F_inv = F.inv()
    S = F_inv * P
    
    # 展开成一维列表以便进行全局优化
    P_list = [P[i, j] for i in range(3) for j in range(3)]
    S_list = [S[i, j] for i in range(3) for j in range(3)]
    
    # 7. 【核心魔法】对 PK1 和 PK2 进行全局公共子表达式消除
    # 这样 P 和 S 共享的中间变量（如 J, F的伴随矩阵等）只会被计算一次
    sub_exprs, simplified = sp.cse([P_list, S_list])
    simplified_P, simplified_S = simplified
    
    # ================= 打印 C++ 代码 =================
    print("#include <cmath>\n")
    print("void compute_neo_hookean_stress(const double* F, double mu, double bulk, double* P, double* S) {")
    
    for var, expr in sub_exprs:
        cpp_expr = sp.ccode(expr)
        print(f"    double {var} = {cpp_expr};")
        
    print("\n    // 赋值给 PK1 (P)")
    for i in range(9):
        print(f"    if (P) P[{i}] = {sp.ccode(simplified_P[i])};")
        
    print("\n    // 赋值给 PK2 (S)")
    for i in range(9):
        print(f"    if (S) S[{i}] = {sp.ccode(simplified_S[i])};")
        
    print("}")

if __name__ == "__main__":
    generate_pk1_pk2_cpp()
```

## 阶段三：C++ 求解器集成 (Zero-Dependency Integration)

**目标：** 将生成的 C++ 代码无缝接入现有的有限元求解器、物理引擎或相关工程项目中。

**操作步骤：**

1. **源码引入：** 将 SymPy 生成的 C 函数（如 `compute_neo_hookean_stress.cpp`）直接保存到工程的源码目录中。
2. **构建系统：** 在 CMakeLists.txt 中添加该文件。代码仅依赖 `<cmath>` 标准库，无任何第三方依赖（无需 TensorFlow, IREE, 或 XLA 库）。
